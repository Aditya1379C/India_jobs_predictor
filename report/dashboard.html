What Drives Your Salary
Insights from feature importance and salary data
  • Company matters most: Which company you work at is the strongest salary predictor, more than experience or skills.
  • Experience is second: Each additional year of experience correlates with meaningful salary growth, especially in 0–5 year range.
  • Role titles diverge: "ML Engineer" and "Data Architect" roles command 2–3× the salary of entry-level analyst roles.
  • City premium is real: Bangalore and Hyderabad listings pay ~20% more than the national average in comparable roles.
  • Skills are the wildcard: AWS, ML, and AI skills add a measurable salary premium even controlling for role.

Key Insights & Conclusions
Market Takeaways
Synthesised from all scraped listings
  • Bangalore leads hiring: More active listings than any other city, cementing its status as India's primary tech hub.
  • Java Developer is most listed: The single highest-demand role across all listings in the dataset.
  • 3281 distinct roles fetched: The India tech market is diverse, spanning Data, ML, Dev, and PM roles.
  • Python is universal: Required in data, ML, backend, and automation roles alike. The safest skill to invest in.
  • Tier-1 cities dominate: Bangalore, Hyderabad, Pune, and Mumbai account for the majority of listings.
About This Project
End-to-end Python pipeline · API-powered
  • Data collection: Live listings fetched from Adzuna API across 18 keywords × 6 cities. 2 pages/keyword = 216 calls/day, within the free tier limit.
  • Data pipeline: Raw API response → Pandas → SQLite via a modular ETL pipeline with salary/experience parsing, imputation, and history-preserving upsert.
  • ML model: XGBoost with LOO target encoding, company tier feature (FAANG/MNC/Indian IT/Startup), 5-fold CV, and log-transformed salary target. R² 0.205 on test set.
  • Salary predictor: Lookup tables from training data embedded in this HTML. Works fully offline — no server needed for this section.
  • Daily pipeline: GitHub Actions scrapes at 11 AM IST, then automatically retrains and regenerates this dashboard, committing the results back to the repo. Just git pull to get the latest.
